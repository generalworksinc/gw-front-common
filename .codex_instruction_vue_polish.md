AGENTS.md の規約に従うこと。

## V-1: useNotification を resolvePinia 経由に統一

`vue/features/notification/notificationStore.ts` は素の defineStore を export しているため、
setup コンテキスト外（サービス層・コールバック）で呼ぶと active な pinia が無いタイミングで例外になる。
`vue/features/loading/loadingStore.ts` の useLoading と同じパターンに揃える:

```ts
const useNotificationPinia = defineStore('notification', { ...現状の定義そのまま... });

// 明示的に pinia の受け渡しを必須化（setup外でも setPinia 済みなら安全に呼べる）
export function useNotification() {
	return useNotificationPinia(resolvePinia());
}
```

シグネチャは `useNotification()`（引数なし）のまま。export 名も不変。

## V-2: レガシー互換APIに @deprecated を付与

`vue/features/loading/loadingStore.ts` の `LOADING()` / `NOT_LOADING()` に JSDoc で
`@deprecated startLoading()/stopLoading() を使うこと。次のメジャーバージョンで削除予定。` を付ける（削除はしない）。
solid 側 loadingStore にも同種のレガシー別名があれば同様に付与（なければ何もしない）。

## V-3: vue Notifications に position プロパティを追加（solid とのパリティ）

solid 版 `solid/features/notification/components/Notifications.tsx` は
`position?: 'top-right' | 'top-left' | 'bottom-right' | 'bottom-left'` prop を受け取り、
コンテナに `position-${position}` クラスを追加する。vue 版にも同じ機能を追加する:

1. `vue/features/notification/notificationStore.ts` に型を追加・export（solid と同名）:

```ts
export type NotificationPosition = 'top-right' | 'top-left' | 'bottom-right' | 'bottom-left';
```

2. `vue/features/notification/components/Notifications.vue` に `position?: NotificationPosition` prop を追加し、
   コンテナ div のクラスを solid と同じ構成にする:

```html
<div class="z-50 position-top-right default-position-style-top-right"
     :class="position ? `position-${position}` : ''">
```

prop 未指定時の出力クラスは現状と完全に同一であること（既存アプリの見た目が変わらない）。
